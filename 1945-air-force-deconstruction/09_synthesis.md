# 9. Synthesis

## 9.1 Key Strengths

1. Simple combat core with low input friction: one-thumb drag and auto-fire make the game easy to start.
2. Strong action-reward-investment loop: short runs feed immediate upgrade decisions.
3. Layered metagame: Aircraft, Wingman, Device, upgrades, events, quests, and PvP-like modes extend retention beyond the arcade core.
4. Reusable gameplay format: campaign, daily missions, multiplayer, and events can reuse the same shooting foundation with different goals/rewards.
5. Clear F2P monetization surfaces: ads, gems, VIP, packs, containers/gacha, and attempt gates attach to natural player needs.

## 9.2 Key Weaknesses

1. Visual readability can degrade when bullets, explosions, loot, and player shots overlap.
2. New player complexity risk: years of added systems can overwhelm if unlock pacing is not carefully staged.
3. UI consistency issue: current notes mention star/% mismatch between in-level and dashboard progression display. Needs current verification.
4. Motivation gap: the game enters action quickly but has little contextual narrative hook.
5. Fairness risk: progression gates can feel like paywalls if earn/spend rates spike too sharply.

## 9.3 Improvement Proposals

### Proposal 1: Improve Combat Readability

- Observation: Dense combat creates excitement but can hide lethal bullets.
- Problem: If players cannot understand why they died, frustration feels unfair.
- Proposal: Increase enemy bullet contrast, reduce overlapping explosion opacity near player, and keep pickups visually distinct from bullets.
- Risk: Too much cleanup may reduce spectacle and power fantasy.

### Proposal 2: Clarify Squadron Onboarding

- Observation: Aircraft/Wingman/Device are core to long-term build depth.
- Problem: If introduced too quickly, they look like UI clutter instead of meaningful unit roles.
- Proposal: Introduce each playable unit with a clear combat demonstration: Aircraft as main weapon, Wingman as coverage, Device as utility/burst.
- Risk: Too much tutorial text slows down the arcade-first opening.

### Proposal 3: Make Progression Gates More Legible

- Observation: Upgrade, promote, modules, stars, tier, and certify can blur together.
- Problem: Players may know they are blocked but not know the next best action.
- Proposal: Add a direct "needed next" path from failed level/result screen to the exact resource/unit upgrade needed.
- Risk: Over-directing can reduce player agency and discovery.
